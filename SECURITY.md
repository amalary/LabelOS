# Security

Security issues should be handled privately and promptly.

## Reporting A Vulnerability

Do not open a public issue for suspected vulnerabilities. Contact the project maintainers through the approved private security channel for this repository.

Include:

- A clear description of the issue.
- Steps to reproduce when safe to share.
- Affected components or files.
- Any known impact or exposure.

## Secrets And Sensitive Data

- Do not commit secrets, credentials, private keys, tokens, or production data.
- Use `.env.example` for placeholders only.
- Rotate any credential that may have been exposed.

## Supported Versions

The project is pre-release. Security support expectations will be documented before production use.
